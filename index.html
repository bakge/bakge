Bakge

Bakersfield Game Engine is a library and development kit designed to simplify programmatic approaches to game, game engine and graphical application development. The SDK provides compiled libraries and runtime binaries for select platforms and compilers with which you can develop, maintain and build your own project. The GDK contains a ready-made engine and tool-chain for developing 2D and 3D games using Lua 5.1.

Develop

Bakge is still unreleased and the GDK engine and tools are still largely undeveloped. The library's public API reference is created using Doxygen, and is provided with all SDK distributions. Additional documentation in the form of examples or tutorials will also be provided with the API reference.

Build

To build Bakge you must first install CMake 2.6+ and git. CMake is a cross-platform build system, and git is a powerful distributed version control system.

Fork & Clone

First, fork Bakge, then clone it to your local machine with the following commands.

$ cd repo_root
$ git clone https://github.com/yourusername/bakge
$ cd bakge

Code

If you want to fix a bug or add a feature, it is best if you create a new branch off of palistov/develop. This way it can easily be tested against the rest of the most recent features or bug fixes. You will want to keep palistov/develop updated in your repository, so you can merge recent changes into your own feature branches if necessary. Run these commands to keep up to date with Bakge's develop branch.

$ git remote add palistov https://github.com/palistov/bakge    # Run ONCE
$ git fetch palistov develop

Once you have the most recent develop branch, check it out and create your own local branch in your repository so you can keep track of it. Remember to git fetch palistov develop often to keep it up to date!

$ git checkout palistov/develop
$ git checkout -b palistov-develop

Now you can create your feature branch and start coding in your newest feature or bug fix. Once you have created your own local branch of palistov/develop, keep it up to date by fetching and pulling with git pull palistov develop while on your palistov-develop branch.

$ git checkout -b my-feature

Contribute

Once your awesome new feature or much-needed bug fix is complete, submit a pull request via Github. Most of the time the pull request won't be merged through Github itself, but it will be cloned to the official repository and merged in.
